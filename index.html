MindMap of List of Tools in course

LINUX

ABOUT

Linux is a well known OS that runs on the server-side. Though there are many vendors of Linux OS, Yet RedHat family OS comes with more stability and is so widely used. From the RedHat family, CentOS is one of the widely used ones.

LIST OF TOPICS

1. Introduction to Linux OS, Command Syntaxes, Hardware and Software information review.
2. Files: Manage files like list, create, delete, copy & rename files
3. Directories: Directory Structure, Navigate Directories, Manage Directories like list, create, delete, copy & rename dir and Move Dir & Files.
4. Editor: VIM editor, Discuss three modes which are ESC, INSERT & COLON Modes.
5. Utilities: Commands like finding files, Download files from internet & Pipes.
6. Administration Topics: Process Management, User Management with SUDO, Package Management, Service Management, Permission Management & Network informative commands.
7. Project Setup: Using the knowledge of Linux, we will setup the project end to end manually with the given manual steps using Linux VMs in AWS Cloud.

OUTCOME

By end of this topic you will know how to manage and work on Linux OS that run on AWS Cloud.

GIT REPOS

ABOUT

GIT is the most widely used VCS utility by most of the modern development projects, Since it has its own advantages over other tools. Services like GitHub makes even the things much simpler in using these tools.

LIST OF TOPICS

1. GitHub: Start Creating account and repositories in GitHub.
2. CLONE: Clone the repositories in local systems using editor from UI and as well as from CLI.
3. COMMIT: Changes to be committed to repositories over editor and CLI.
4. PUSH: Authenticating ways to GitHub to push the Code to Central Repositories
5. PULL: Pull the code changes to the remote servers to execute the code on the server side.
6. MERGE: In cases we choose to process the approach of editing the code then we usually go with creating Branches and merge the code with main code after the process of approvals.
7. TAGS: Git Tags are the most widely used strategy among the industry in release management and we do perform creating tags and aligning the whole CICD process based on the tags.

OUTCOME

By end of this topic you will know how to manage code and commit the local changes and push the changes to remote repositories. Also as a standard practice we pull the code on servers and run those on server rather than making the code on servers.

SHELL SCRIPTING

ABOUT

Shell is the default in Linux and hence most of the automation goes on with Shell Scripting in Linux OS. This is very simple yet very powerful when it comes to automation. 

LIST OF TOPICS

1. SHELL TYPES, Types of shell and why do we choose BASH.
2. SHE-BANG & COMMENTS: Use of She-bang in scripts and followed with how to write comments.
3. PRINT OUTPUT: How to print text while running the shells scripts. Colorizing output to make the scripts look much effective.
4. VARIABLES: Deal with basic variables, then followed with properties of the variables, then followed with dynamic variable declarations.
5. INPUTS: Collecting the inputs from the user to run the scripts.
6. FUNCTIONS: Deal with functions to keep the CODE DRY.
7. CONDITIONS & LOOPS: Logic control of scripts using if condition and iterative way of executing things using for and while loop.
8. MISC: Redirectors, Exit Status and their usage in scripts
9. PROJECT SETUP USING SHELL SCRIPTS: Using the knowledge of the topics which were learned we go with implementation in project.

OUTCOME

By end of this topic you will know how to write shell scripting and write the code in standard ways.

ANSIBLE

ABOUT

Ansible is currently the leading Configuration Management tool in the market usage. Though it is simple and most powerful one with a lot of loaded collection with modules.

LIST OF TOPICS

1. Configuration Management: Introduction to CM, Evolution of CM, Advantages of CM, How Ansible is leader, Ansible Installation
2. Inventory: Ansible Inventory and how best it can be setup as per need.
3. Ansible with SSH: Ansible using SSH Password, then SSH Keys, Overview of LDAP or AD, Ansible Configuration file
4. Ad-Hoc: Executing Ansible Ad-Hoc commands
5. Playbooks: Need of Playbook, Introduction to YAML, Playbook Layout, Simple Playbook that connects to the nodes.
6. Variables: Different ways of parsing variables.
7. Roles: Need of Roles, Role structure, Playbooks on roles
8. Misc: Tags, Filters, Handlers Loops, Vault.
9. Ansible Pull: Need of pull
10. Ansible ATX: UI Management tool for Ansible.

OUTCOME

By end of this topics we learn the options in Ansible and then we will implement the project in Ansible to support the way it requires in realtime for multiple environments like "DEV", "PROD".

JENKINS

ABOUT

Jenkins is well known for CI tool, Yet it can be used as automation tool as well. Hence we will try to adopt this tool for automating the work which we did so far with the help of Jenkins pipelines.

LIST OF TOPICS

1. Introduction: Introduction to Jenkins, Installation & Configuration
2. Jobs: FreeStyle vs Pipeline (Declarative vs Scripted).
3. Jenkinsfile: Discuss the different options in Jenkinsfile.
4. Seed Jobs: The way for creating automated jobs in Jenkins by using seed jobs.
5. Plugins: Different plugins that we use for the requirements of automating our efforts.
6. Shared Libraries: When we start using the pipelines extensively then we will start seeing the need of keeping the code DRY and that is where the Shared Libraries in Jenkins comes into picture. Also this is widely used in enterprise market.

CI is one of the practice in the modern development. It helps developers to identify the issues in much early before releasing the code. DevOps engineers usually try to make these configurations and help developers to stabilize the process.


1. CI Pipelines: Introduction to CI & Pipelines steps involved in general.
2. SonarQube: Setup sonarqube application and scan the code and report it.
3. Nexus: Use this to archive the artifacts in central place.
4. Selenium: How to do the automated tests when there is a new application has been about to release
5. Builds: How to make the compile the code and make the builds.
6. Release Strategy: Different best practices and its outcomes to enhance these pipelines to Continuous Delivery and then to Continuous Deployment.
In this strategy we discuss about branching, tagging and merging the code in Git repos.

OUTCOME

By end of this topic you will know how the flow works for the development, CI Pipelines with Jenkins. Also some of the Jenkins best practices like DRY will be implemented with SEED jobs and Shared Libraries.

TERRAFORM

ABOUT

Infrastructure as Code(IaC) is also a key and foundation for DevOps practices since modern applications are tying to achieve Immutable Infrastructure. Early days infra was the one of the most manual effort involved in the whole process of releasing the applications, But Terraform has solved the problem by providing a common syntax that works with multi-cloud. 

LIST OF TOPICS

1. IaC: Introduction to IaC (Infrastructure as Code) tool and need for it. What Terraform is helping us to achieve.
2. HCL: Terraform DSL which is HCL v2 (Hashicorp Configuration Language). Introduction to it, Some basics of how it looks like.
3. Introduction: Terraform Files, Terraform Commands.
4. Outputs: Terraform outputs and different ways of using it with root module and sub module.
5. Variables: Different ways of parsing variables to Terraform and its precedence and best practices. Also followed with Data types & Types of Variables in Terraform.
6. Terraform Core Resources: Providers, Resources, Provisioners
7. Modules: Need for modules and how it helps to keep the code DRY.
8. Logical Statements: Terraform Loops & Conditions.
9. Functions: In Built functions of Terraform.

OUTCOME

By end of this topic we undergo completing our project with Terraform extensively all the topics in it. Majorly we use terraform to create infra for both Mutable and Immutable approaches and it involves in CICD deployments.

PROMETHEUS & GRAFANA

ABOUT

Monitoring is the key factor in the operations world. Also getting the insights of the application usage and performance is also a very important factor helps you in proactively monitor and resolve the issues of the application platform. Tools like Nagios or Zabbix helps in blackbox monitoring but not whitebox. Hence tools like Prometheus are more demand in market.

LIST OF TOPICS

1. Introduction: Basics of Monitoring and how Prometheus is different than others.
2. Installation & Configuration: Prometheus Monitoring Setup, Pull Metrics, Dynamic Nodes Setup with AWS.
3. Functions: Inbuilt functions of Prometheus and setup custom record rules.
4. Grafana: Create Dashboards from Prometheus Data Source.
5. Alert Manager: Configure Alerts in Prometheus Alert Manager and report any kind of application issue to email.

OUTCOME

By end of this topic we undergo learning Prometheus and we monitor our services using this tool and grab Metrics in Grafana.

ELK

ABOUT

Log Aggregation is a very important piece to get the analytics of the application with respect to performance and usage of it. So all the logs of applications from all the instances are really very important and needs to be placed in some central system. So tools like ELK gives log aggregation in better way and it is open-source as well.

LIST OF TOPICS

1. Log Aggregation: Introduction to log aggregation and Importance of it in Business.
2. Components: Understand the architecture of ELK, Components Involved in it.
3. Logs: Understand Structured and Unstructured logs and use components like Filebeat & Logstash to handle them.
4. Kibana: Collect log indices and create patters in Kibana, Choose data types for the logs coming in, Dashboards & Data Visualisation in Kibana.

OUTCOME

By end of this topic you will know how to collect logs and make some analytics on top of that using Kibana Dashboard and understand the performance of our project application. 

DOCKER

ABOUT

Clearly Docker major player in the container field, And Docker is only one form of container technology. So we talk about how to manage containers and focus on making Docker Imaging

LIST OF TOPICS

1. Docker: Introduction to Containers & Anatomy of Docker, Docker Echo System, Docker container options
2. Docker Imaging: Making Docker Images & Setup project component Images.

OUTCOME

By end of this topic you will know how to manage docker containers and make images of docker as per the need of the project.

KUBERNETES
&
HELM

ABOUT

Kubernetes is a portable, extensible, open-source platform for managing containerised workloads and services, that facilitates both declarative configuration and automation. It has a large, rapidly growing ecosystem. Kubernetes services, support, and tools are widely available. Helm is a package manager for Kubernetes

LIST OF TOPICS

1. Kubernetes Introduction: Need of an Orchestration tool and why Kubernetes is leading, Echo System of Kubernetes
2. Installation: Setup a single node cluster for lab purpose and EKS cluster for full project.
2. Kubernetes Resources: Pods, Services, ReplicaSets, Deployments, DaemonSets, StatefulSets, Secrets, ConfigMaps. HPA (Auto scaling), Resources of Pods.
3. Monitoring: Enabling Prometheus & Grafana monitoring.
4. Log Aggregation: Enabling Log aggregation on Kubernetes Cluster
5. Helm Charts: Use of Helm Charts, Charts for project. Deployment of charts using Jenkins.
6. FULL CICD: Full CICD on Kubernetes using rolling updates.

OUTCOME

By end of this topic you will know how to manage containers in Kubernetes cluster, Scale them, Release applications in much faster way.
